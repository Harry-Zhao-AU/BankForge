# Phase 03 Plan 04: Observability + Kiali + STRICT mTLS

**Goal:** Full observability stack running in the bankforge namespace (OTel Collector, Jaeger, Prometheus, Loki, Grafana) + Kiali in istio-system, then switch Istio to STRICT mTLS. validate-observability-k8s.sh passes with zero SKIPs.

**Why before 03-03 (Kong/Keycloak):** You need traces in Jaeger and a service graph in Kiali before adding JWT auth complexity. Debugging Kong issues blind (no traces) is expensive.
