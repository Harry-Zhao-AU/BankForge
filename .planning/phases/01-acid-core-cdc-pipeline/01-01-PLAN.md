**TransferStateMachine.java** (package `au.com.bankforge.common.statemachine`):
- Spring `@Component`
- Static `Map<TransferState, Map<TransferEvent, TransferState>> TRANSITIONS` using `Map.of()`:
  - PENDING + INITIATE -> PAYMENT_PROCESSING
  - PAYMENT_PROCESSING + PROCESS -> PAYMENT_PROCESSING (idempotent re-entry for retries)
  - PAYMENT_PROCESSING + PAYMENT_COMPLETE -> PAYMENT_DONE
  - PAYMENT_PROCESSING + FAIL -> COMPENSATING
  - PAYMENT_DONE + POST -> POSTING
  - POSTING + CONFIRM -> CONFIRMED
  - POSTING + FAIL -> COMPENSATING
  - COMPENSATING + COMPENSATE -> CANCELLED
- Method: `public TransferState transition(TransferState current, TransferEvent event)` that looks up the transition and throws `InvalidStateTransitionException` if not found.

**InvalidStateTransitionException.java** (same package):
- Extends `RuntimeException`
- Constructor: `InvalidStateTransitionException(TransferState current, TransferEvent event)` with message `"Invalid transition: cannot apply " + event + " to state " + current`

**ValidBsb.java** (package `au.com.bankforge.common.validation`):
- Jakarta Bean Validation constraint annotation: `@Constraint(validatedBy = BsbValidator.class)`
- `@Target({ElementType.FIELD, ElementType.PARAMETER})`, `@Retention(RetentionPolicy.RUNTIME)`
- Default message: `"Invalid BSB format. Must be NNN-NNN (e.g., 012-345)"`

**BsbValidator.java** (same package):
- Implements `ConstraintValidator<ValidBsb, String>`
- Pattern: `^\\d{3}-\\d{3}$`
- `isValid()` returns false for null (null rejection), true only for exact NNN-NNN match

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-1-04 | Information Disclosure | PostgreSQL wal_level=logical | accept | Logical WAL exposes more data than default but this is required for Debezium CDC in Phase 1.1. Local-only network. |
| T-1-05 | Denial of Service | Redis (no auth) | accept | Local dev only; no external network exposure. Phase 3 adds Kubernetes NetworkPolicy. |
</threat_model>
