    private final AccountRepository accountRepository;

            .accountNumber(request.accountNumber())
            .accountName(request.accountName())
            .balance(request.initialBalance() != null ? request.initialBalance() : BigDecimal.ZERO)

    @Transactional(readOnly = true)
    public List<TransferHistoryResponse> getTransferHistory(UUID accountId) {

            .orElseThrow(() -> new AccountNotFoundException(accountId));

**TransferService.java** (package `au.com.bankforge.account.service`):
This is the CRITICAL ACID method per CORE-02 and TXNS-01.

```java
@Service
@RequiredArgsConstructor
public class TransferService {
    private final AccountRepository accountRepository;
    private final OutboxEventRepository outboxEventRepository;
    private final ObjectMapper objectMapper;

    @Transactional
    public TransferResponse executeTransfer(TransferRequest request) {
        UUID fromId = request.fromAccountId();
        UUID toId = request.toAccountId();
        BigDecimal amount = request.amount();

        // DEADLOCK PREVENTION: Always lock lower UUID first
        UUID first  = fromId.compareTo(toId) < 0 ? fromId : toId;
        UUID second = fromId.compareTo(toId) < 0 ? toId   : fromId;

        Account lockFirst  = accountRepository.findByIdForUpdate(first)
            .orElseThrow(() -> new AccountNotFoundException(first));
        Account lockSecond = accountRepository.findByIdForUpdate(second)
            .orElseThrow(() -> new AccountNotFoundException(second));

        Account debit  = lockFirst.getId().equals(fromId)  ? lockFirst : lockSecond;
        Account credit = lockFirst.getId().equals(fromId) ? lockSecond : lockFirst;

        // Balance check AFTER lock acquisition
        if (debit.getBalance().compareTo(amount) < 0) {
            throw new InsufficientFundsException(fromId, amount);
        }

        // Atomic balance update
        debit.setBalance(debit.getBalance().subtract(amount));
        credit.setBalance(credit.getBalance().add(amount));

        // Outbox write — SAME transaction
        UUID transferId = UUID.randomUUID();
        OutboxEvent outbox = OutboxEvent.builder()
            .aggregatetype("Transfer")
            .aggregateid(transferId.toString())
            .type("TransferInitiated")
            .payload(serializePayload(transferId, fromId, toId, amount))
            .build();
        outboxEventRepository.save(outbox);

        return new TransferResponse(transferId, fromId, toId, amount, "COMPLETED", Instant.now());
    }

    private String serializePayload(UUID transferId, UUID fromId, UUID toId, BigDecimal amount) {
        try {
            TransferCreatedEvent event = new TransferCreatedEvent(
                transferId, fromId, toId, amount, "COMPLETED", Instant.now());
            return objectMapper.writeValueAsString(event);
        } catch (Exception e) {
            throw new RuntimeException("Failed to serialize outbox payload", e);
        }
    }
}
```

Key points:
- `@Transactional` wraps debit + credit + outbox write in ONE transaction per TXNS-01
- Lock ordering: `min(fromId, toId)` locked first to prevent deadlocks
- `findByIdForUpdate()` uses `PESSIMISTIC_WRITE` per D-10
- Balance check AFTER lock — not before — to prevent race condition
- `ObjectMapper` is the auto-configured Spring bean. **CRITICAL: Jackson 3 import required.** Use `tools.jackson.databind.ObjectMapper` (Jackson 3, bundled with Spring Boot 4.0.5). Do NOT import `com.fasterxml.jackson.databind.ObjectMapper` — that is the Jackson 2 package which will not compile under Spring Boot 4.

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-1-01 | Tampering | TransferService.executeTransfer | mitigate | PESSIMISTIC_WRITE locks prevent concurrent balance manipulation. Lock ordering (ascending UUID) prevents deadlocks. @Transactional ensures atomicity. |
| T-1-02 | Input Validation | AccountController / TransferController | mitigate | Jakarta @Valid on all request bodies. @ValidBsb for BSB format. @DecimalMin("0.01") on amounts. @NotNull on required fields. GlobalExceptionHandler returns 400 for validation failures. |
| T-1-03 | Repudiation | TransferService outbox write | mitigate | Every transfer creates an outbox_event row with transfer ID, account IDs, amount, and timestamp in the same ACID transaction. This is an immutable audit trail. |
