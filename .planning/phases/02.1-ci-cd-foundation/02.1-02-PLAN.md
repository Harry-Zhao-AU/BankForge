```yaml
name: CI

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

permissions:
  contents: read
  packages: write

jobs:
  build-common:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up JDK 21
        uses: actions/setup-java@v4
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: 'maven'

      - name: Build and install common module
        run: mvn -B install -pl common -am -Dmaven.test.skip=true

  build-test-push:
    needs: build-common
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        service:
          - account-service
          - payment-service
          - ledger-service
          - notification-service
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up JDK 21
        uses: actions/setup-java@v4
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: 'maven'

      - name: Test ${{ matrix.service }}
        run: mvn -B test -pl ${{ matrix.service }}

      - name: Log in to ghcr.io
        if: github.event_name == 'push' && github.ref == 'refs/heads/master'
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata for ${{ matrix.service }}
        if: github.event_name == 'push' && github.ref == 'refs/heads/master'
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository_owner }}/bankforge-${{ matrix.service }}
          tags: |
            type=raw,value=latest
            type=sha,format=short

      - name: Build and push ${{ matrix.service }} image
        if: github.event_name == 'push' && github.ref == 'refs/heads/master'
        uses: docker/build-push-action@v6
        with:
          context: .
          file: ${{ matrix.service }}/Dockerfile
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
```

Create 4 Dockerfiles, one in each service directory. Each Dockerfile follows the same multi-stage pattern but references its own service name. The build context is the repo root (`.`), so all paths are relative to root.

**account-service/Dockerfile:**
```dockerfile
# Multi-stage build: JDK 21 builder + JRE 21 runtime
# Build context must be repo root (docker build -f account-service/Dockerfile .)

# ---- Build stage ----
FROM eclipse-temurin:21-jdk-jammy AS builder
WORKDIR /build

# Copy POMs first for dependency layer caching
COPY pom.xml ./
COPY common/pom.xml common/
COPY account-service/pom.xml account-service/

# Pre-fetch dependencies (cached until pom.xml changes)
RUN mvn -B dependency:go-offline -pl account-service -am -Dmaven.test.skip=true || true

# Copy source
COPY common/src common/src
COPY account-service/src account-service/src

# Package (tests run in CI job, not in docker build)
RUN mvn -B package -pl account-service -am -Dmaven.test.skip=true

# ---- Runtime stage ----
FROM eclipse-temurin:21-jre-jammy AS runtime
WORKDIR /app

RUN addgroup --system appgroup && adduser --system --ingroup appgroup appuser

COPY --from=builder /build/account-service/target/*.jar app.jar
RUN chown appuser:appgroup app.jar

USER appuser
EXPOSE 8080

ENTRYPOINT ["java", "-XX:MaxRAMPercentage=75.0", "-jar", "app.jar"]
```

**payment-service/Dockerfile** — identical structure, replace all occurrences of `account-service` with `payment-service`:
- `COPY payment-service/pom.xml payment-service/`
- `RUN mvn -B dependency:go-offline -pl payment-service -am ...`
- `COPY payment-service/src payment-service/src`
- `RUN mvn -B package -pl payment-service -am ...`
- `COPY --from=builder /build/payment-service/target/*.jar app.jar`

**ledger-service/Dockerfile** — identical structure, replace with `ledger-service`.

**notification-service/Dockerfile** — identical structure, replace with `notification-service`.

Per D-02: Multi-stage build with eclipse-temurin:21-jdk-jammy (build) and eclipse-temurin:21-jre-jammy (runtime).
Per D-03: Standard `docker build` compatible (no Jib, no CNB).

IMPORTANT details:
- `|| true` on `dependency:go-offline` prevents build failure if some dependencies cannot be resolved offline (common with SNAPSHOT deps). The actual `package` step will fetch anything missing.
- `-Dmaven.test.skip=true` skips both test compilation and execution (per CLAUDE.md convention).
- Non-root `appuser` in runtime stage (ASVS L1 security practice).
- `EXPOSE 8080` documents the Spring Boot default port.
- `-XX:MaxRAMPercentage=75.0` limits JVM heap to 75% of container memory (prevents OOMKill in K8s with memory limits in Phase 3).
- The `*.jar` glob in COPY works because `spring-boot-maven-plugin` produces exactly one executable JAR in target/.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR author -> CI workflow | Untrusted code from fork PRs runs on GitHub-hosted runners |
| CI workflow -> ghcr.io | GITHUB_TOKEN authenticates image push; scoped to packages:write |
| Dockerfile -> base images | Eclipse Temurin images from Docker Hub (trusted, official) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-02.1-02 | Elevation of Privilege | ghcr.io image push | mitigate | Image push gated to `github.event_name == 'push' && github.ref == 'refs/heads/master'`; GITHUB_TOKEN has `packages: write` only; no PATs stored as secrets (per D-11) |
| T-02.1-03 | Tampering | Dockerfile base images | accept | Eclipse Temurin images are official Adoptium builds on Docker Hub; pinning to `21-jdk-jammy` and `21-jre-jammy` tags (OS-version pinned but not SHA-pinned). Acceptable risk for a learning project; production would pin image digests. |
| T-02.1-04 | Tampering | Workflow injection | mitigate | No `${{ github.event.head_commit.message }}`, `${{ github.event.pull_request.title }}`, or other user-controlled expressions interpolated into `run:` blocks. All dynamic values are passed through action `with:` parameters which are not shell-interpreted. |
| T-02.1-05 | Information Disclosure | CI logs | accept | GITHUB_TOKEN is auto-masked in logs by GitHub Actions. No other secrets are used. Maven build output may contain dependency URLs (public) and test names (non-sensitive). |
| T-02.1-06 | Elevation of Privilege | Dockerfile runtime user | mitigate | Runtime stage runs as non-root `appuser` via `USER appuser` directive. Builder stage runs as root (standard for build stages; no runtime exposure). |
</threat_model>
