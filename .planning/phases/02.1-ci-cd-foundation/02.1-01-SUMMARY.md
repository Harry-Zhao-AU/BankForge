## Decisions Made

- Maven `<activation><os><family>Windows</family></os></activation>` used instead of `-P podman-windows` explicit activation — the profile activates automatically on Windows dev machines without developer needing to remember a flag, and does NOT activate on Linux CI runners.
- Comment in each POM updated to explain the two-tier design (base = OS-neutral includes; profile = Windows-specific env vars).

## Issues Encountered

**Spring context cache poisoning (pre-existing, out of scope):**
When running the full account-service test suite sequentially, `AccountControllerIT` (5 tests) passes, but `ConcurrentTransferIT` and `TransferServiceIT` fail with "ApplicationContext failure threshold exceeded" — a Spring test context caching issue where a prior context failure poisons the cache for subsequent tests. This is confirmed pre-existing (unrelated to DOCKER_HOST changes) because:
1. `AccountControllerIT` passes — proves profile activates and Testcontainers connects to Podman
2. `TransferServiceIT` passes when run in isolation (`-Dtest=TransferServiceIT`)
3. The failure message is "context failure threshold exceeded", not "Docker not found"

This issue predates this plan and is tracked for a future fix (likely `@DirtiesContext` ordering or context isolation).
