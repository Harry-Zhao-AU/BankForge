## What Was Built

### FAILED Enum State (TransferState.java)
Added `FAILED` as the last value in `TransferState` after `CANCELLED`. Updated Javadoc state diagram to show the `POSTING --> FAILED` path. This state is set directly by the `HungTransferDetector` scheduler (Plan 03), not via FSM transition.

### V3 Flyway Migration (V3__add_ledger_outbox.sql)
Created `ledger_outbox_event` table with Debezium EventRouter-compatible column names (`aggregatetype`, `aggregateid`, `type`, `payload`, `created_at`). Column comments document the Debezium contract and the D-15 header propagation intent.

### LedgerOutboxEvent Entity (LedgerOutboxEvent.java)
JPA entity mapping to `ledger_outbox_event`. Uses `@JdbcTypeCode(SqlTypes.JSON)` on `payload` (Hibernate 7 requirement). `@PrePersist` sets `createdAt = Instant.now()`. Field names match the SQL table column names exactly — Debezium EventRouter reads them verbatim.

### LedgerOutboxEventRepository
Minimal Spring Data JPA repository extending `JpaRepository<LedgerOutboxEvent, UUID>`. Plan 02 will inject this to write outbox rows within the same transaction as ledger entry persistence.

### Debezium Connector (ledger-outbox-connector.json)
Routes CDC events from `public.ledger_outbox_event` to the literal Kafka topic `banking.transfer.confirmed` (not the dynamic `${routedByValue}` pattern used by the account connector — ledger always routes to one fixed topic). Key D-15 Layer 1 configuration:
```json
"transforms.outbox.table.fields.additional.placement": "aggregateid:header:transaction-id"
```
This copies the `aggregateid` column (which contains the `transferId`) into a Kafka record header named `transaction-id` on every message published to `banking.transfer.confirmed`.

### register-connector.sh
Extended to register `bankforge-ledger-outbox-connector` idempotently after the existing account connector. Uses PUT for updates (connector exists) and POST for creates (connector absent), matching the account connector pattern.

### compose.yml
Added `ledger-db: condition: service_healthy` to `debezium-connect.depends_on`. Debezium now waits for ledger-db to be healthy before starting, preventing connector registration failures against an unreachable database.
