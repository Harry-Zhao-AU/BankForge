## What Was Built

### LedgerEventListener Refactor (outbox pattern)

Removed `KafkaTemplate` injection and both `kafkaTemplate.send()` calls. Replaced with `LedgerOutboxEventRepository` injection and a single `ledgerOutboxEventRepository.save(outboxRow)` call within the existing `@Transactional` scope. The outbox row uses the Debezium EventRouter-compatible schema (`aggregatetype`, `aggregateid`, `type`, `payload`) from Plan 01.

**Idempotency path** is now a true no-op: when `existsByTransferId(transferId)` is true, the method logs and returns immediately with no writes and no Kafka publish. This is correct because Debezium already published the original outbox row — replaying the Kafka send would cause duplicate confirmation events downstream.

**OTel Baggage** (D-15, D-16): After the idempotency guard passes, `Baggage.current().toBuilder().put("banking.transaction.id", transferId.toString())` is set and all debit/credit/outbox writes occur within `try (Scope scope = baggage.makeCurrent())`.

### KafkaConfig Refactor

Removed the `kafkaTransactionManager` bean entirely. Removed `ContainerProperties.EOSMode.V2` and `KafkaTransactionManager` from `kafkaListenerContainerFactory`. Added `KafkaTemplate<String, String> kafkaTemplate` parameter (auto-configured, non-transactional after application.yml cleanup) and wired `DeadLetterPublishingRecoverer` routing to `banking.transfer.events.DLT` with partition preservation. ExponentialBackOff kept as-is (1s initial, 2x multiplier, 30s max elapsed — D-11).

### application.yml Cleanup

Removed two EOS-related properties:
- `spring.kafka.consumer.isolation-level: read_committed` — default `read_uncommitted` is correct for Debezium-sourced events
- `spring.kafka.producer.transaction-id-prefix: ledger-tx-` — removal makes the auto-configured `KafkaTemplate` non-transactional, required by `DeadLetterPublishingRecoverer`

### DltHandler (new file)

Created `DltHandler.java` consuming from `banking.transfer.events.DLT` with `groupId = "ledger-service-dlt"` (separate consumer group from main listener). Logs structured ERROR with topic, partition, offset, key, value, and all DLT headers (`kafka_original-topic`, `kafka_original-partition`, `kafka_original-offset`, `kafka_exception-fqcn`, `kafka_exception-message`). Contains no `kafkaTemplate.send()` or any publish call — read-only handler per T-01.2-04 mitigate disposition.

### PaymentController Enhancement

Added `.header("X-Transaction-Id", response.transferId().toString())` to the `initiateTransfer` return statement. Uses the record accessor `transferId()` from `InitiateTransferResponse`. No other changes.

### PaymentService OTel Baggage (D-15 Layer 2)

Added `io.opentelemetry.api.baggage.Baggage` and `io.opentelemetry.context.Scope` imports. After `transferStateService.createAndStart(request)` succeeds (and before `incrementTransferInitiated`), sets `Baggage.current().toBuilder().put("banking.transaction.id", transfer.getId().toString()).build()`. Wraps steps 3-6 (account-service HTTP call, advanceToPosting, error handling, reversal, cancel, buildAndCacheResponse) in `try (Scope scope = baggage.makeCurrent())`. Idempotency early-return paths (Redis cache hit and DataIntegrityViolationException) intentionally do not set baggage — they are replay paths where the original execution already propagated baggage.

- T-01.2-07 (retry DoS): `maxElapsedTime=30000ms` caps retry duration before DLT routing
