## Deviations from Plan

### Auto-fixed Issues

**1. WSL2 networking requires 2 workarounds not in original plan**
- **Found during:** Task 02 spike execution
- **Issue 1:** aardvark-dns fails in WSL2 with "Transport endpoint is not connected" — kind network must use `--disable-dns`
- **Issue 2:** journald log driver causes `podman logs` to return no output — kind cannot detect systemd startup; must use `k8s-file`
- **Fix:** Both workarounds added to script body in `f2616aa`
- **Committed in:** `f2616aa`

**2. Leftover kind network from prior run blocks re-run**
- **Issue:** `podman network rm kind || true` silently swallows errors when network is in use; `podman network create` then fails
- **Fix:** Use `podman network rm --force kind` before re-running the script
- **Note:** This is a one-time pre-run step; script is idempotent once the network is cleared

## Issues Encountered
