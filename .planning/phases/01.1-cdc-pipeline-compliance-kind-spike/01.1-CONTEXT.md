<decisions>
## Implementation Decisions

### Kafka Setup
- **D-01:** Kafka runs in KRaft mode — no ZooKeeper. `KAFKA_PROCESS_ROLES=broker,controller`. Use Kafka 3.8+ image.
- **D-02:** Outbox + Debezium CDC only — no polling outbox, no dual-write. Debezium reads PostgreSQL WAL directly via logical replication.

### Debezium CDC
- **D-03:** Debezium connector config must include `slot.drop.on.stop=true` to prevent replication slot leaks on container restart.
- **D-04:** PostgreSQL `wal_level=logical` and `max_replication_slots=5` already set in Phase 1 — no DB restart needed.
- **D-05:** Outbox EventRouter SMT (`io.debezium.transforms.outbox.EventRouter`) routes outbox rows to correct Kafka topics by event type.

### Dead Letter Queues
- **D-06:** DLT (Dead Letter Topic) topics configured from Day 1 for all financial Kafka consumers. Silent event loss is unacceptable for financial events. Use Spring Kafka's `DeadLetterPublishingRecoverer` with exponential backoff.

### AUSTRAC Compliance
- **D-07:** AUSTRAC threshold = AUD $10,000. Any transfer ≥ $10,000 produces a structured compliance event with: timestamp, source account ID, destination account ID, amount, transfer ID.
- **D-08:** Compliance events logged as structured JSON to a dedicated audit log (file or separate Kafka topic). Must be queryable for a given transfer ID.

### Kind Networking Spike
- **D-09:** Spike validates `KIND_EXPERIMENTAL_PROVIDER=podman` single-node cluster startup, test pod reachability, and DNS resolution of cluster-internal names from a pod. This is a prerequisite gate for Phase 3.
- **D-10:** If Podman rootless fails with kind, try rootful Podman. Document the result in STATE.md so Phase 3 planning knows the working mode.

### Claude's Discretion
- Kafka topic naming convention (e.g. `banking.outbox.transfers` vs `transfer-events`) — choose a consistent pattern and document it.
- Compose service startup ordering — PostgreSQL → Kafka → Debezium connector registration ordering via `depends_on` + health checks.
- Whether kind spike is a shell script or a Makefile target.

</decisions>

<specifics>
## Specific Ideas

- Debezium runs as a Kafka Connect plugin (not standalone server) — use `confluentinc/cp-kafka-connect` or `debezium/connect` image.
- The Outbox EventRouter SMT is the key config that turns raw CDC rows into domain events on the right topics.
- AUSTRAC logging can be implemented as a Kafka consumer in payment-service or notification-service that watches the transfer topic — no new service needed.
- kind spike can be a standalone shell script `scripts/kind-smoke-test.sh` that runs independently of the main Compose stack.

</specifics>
