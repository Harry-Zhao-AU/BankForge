# Plan 01.1-03: Podman + kind Networking Spike

## Objective

Validate that a kind Kubernetes cluster can start via Podman, deploy a test pod, and resolve cluster-internal DNS. This is a prerequisite gate for Phase 3 (Service Mesh). The spike runs independently of the compose.yml services.

Purpose: FLAG 2 in CLAUDE.md identifies Podman + kind networking as a risk. This spike resolves that risk before Phase 3 planning begins.

Note: This plan covers phase-level Success Criterion 4 ("kind cluster starts with Podman and resolves DNS"). There is no corresponding REQUIREMENTS.md ID for this infrastructure spike — it is a risk-reduction prerequisite, not a functional requirement.

## Must-Haves (Goal-Backward Verification)

- [ ] `kind` binary is installed and executable (verify: `kind version`)
- [ ] `kubectl` binary is installed and executable (verify: `kubectl version --client`)
- [ ] `scripts/kind-smoke-test.sh` exists and is executable
- [ ] Running `scripts/kind-smoke-test.sh` creates a kind cluster, deploys a test pod, resolves DNS, and cleans up
- [ ] Spike passes OR failure mode is documented with workaround in STATE.md
- [ ] kind cluster is deleted after the spike (no lingering resources)

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Rootless Podman causes kind CoreDNS failure (Denial of Service) | HIGH | Script checks Podman rootful mode before starting; fails fast with install instructions if rootless |
| kind cluster left running after spike (Resource Leak) | LOW | Script always runs `kind delete cluster --name bankforge-spike` in cleanup, including on error (trap) |
| WSL2 networking conflict with Windows firewall | LOW | Spike uses cluster-internal DNS only; no host-to-cluster port mapping needed |

## Tasks

### Task 01 — Install kind and kubectl in WSL2
**Wave:** 0 (prerequisite)

**Read first:**
- Research section "Environment Availability" — kind and kubectl are NOT found on PATH; must be installed in WSL2.

**Action:**

The kind and kubectl CLIs must be installed inside the WSL2 distribution that Podman uses. These commands run inside WSL2 (not on the Windows host).

1. **Check if kind is already installed:** Run `kind version 2>/dev/null`. If it returns a version, skip installation. If not found:

Install kind v0.27.0 (latest stable) inside WSL2:
```bash
# Run inside WSL2 (via: wsl -d podman-machine-default -- bash)
# Or from Windows terminal if wsl is the default shell context
curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.27.0/kind-linux-amd64
chmod +x ./kind
sudo mv ./kind /usr/local/bin/kind
kind version
```

If the Podman machine WSL distribution is not directly accessible via `wsl -d`, try:
```bash
podman machine ssh "curl -Lo /tmp/kind https://kind.sigs.k8s.io/dl/v0.27.0/kind-linux-amd64 && chmod +x /tmp/kind && sudo mv /tmp/kind /usr/local/bin/kind && kind version"
```

2. **Check if kubectl is already installed:** Run `kubectl version --client 2>/dev/null`. If it returns a version, skip installation. If not found:

Install kubectl (latest stable) inside WSL2:
```bash
# Same context as kind installation
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl
sudo mv kubectl /usr/local/bin/kubectl
kubectl version --client
```

Or via Podman machine SSH:
```bash
podman machine ssh "curl -LO 'https://dl.k8s.io/release/\$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl' && chmod +x kubectl && sudo mv kubectl /usr/local/bin/kubectl && kubectl version --client"
```

3. **Verify Podman machine is rootful:** Run `podman machine inspect --format '{{.Rootful}}'` (or `podman machine inspect` and check `"Rootful": true`). The machine must be rootful for kind to work. If rootless:
```bash
podman machine stop
podman machine set --rootful
podman machine start
```

**IMPORTANT:** The research confirms the machine is already rootful (`Rootful=true`). This step is a safety check — do NOT change to rootless.

**Acceptance criteria:**
- `kind version` exits 0 and prints a version string
- `kubectl version --client` exits 0 and prints a version string
- `podman machine inspect` shows `Rootful: true` (or equivalent)

### Task 02 — Create and run kind-smoke-test.sh
**Wave:** 1 (depends on Task 01 for kind/kubectl availability)

**Read first:**
- Research section "Pattern 6: kind Spike Script" (reference script structure)
- Research section "Pitfall 6: kind Cluster DNS Fails with Rootless Podman"

**Action:**

Create `scripts/kind-smoke-test.sh` with the following content:

```bash
#!/usr/bin/env bash
# =============================================================================
# kind Networking Spike — Phase 1.1
#
# Validates: kind + Podman rootful mode, single-node cluster creation,
# test pod reachability, and cluster-internal DNS resolution.
#
# Prerequisite gate for Phase 3 (Service Mesh & Auth).
# =============================================================================
set -euo pipefail

CLUSTER_NAME="bankforge-spike"

# Cleanup trap — always delete cluster, even on failure
cleanup() {
    echo ""
    echo "=== Cleanup: Deleting kind cluster '$CLUSTER_NAME' ==="
    KIND_EXPERIMENTAL_PROVIDER=podman kind delete cluster --name "$CLUSTER_NAME" 2>/dev/null || true
    echo "Cleanup complete."
}
trap cleanup EXIT

echo "================================================================"
echo "  BankForge kind + Podman Networking Spike"
echo "================================================================"
echo ""

echo "=== Step 1: Verify prerequisites ==="
echo "Checking kind..."
kind version || { echo "ERROR: kind not found. Install: curl -Lo kind https://kind.sigs.k8s.io/dl/v0.27.0/kind-linux-amd64 && chmod +x kind && sudo mv kind /usr/local/bin/kind"; exit 1; }

echo "Checking kubectl..."
kubectl version --client || { echo "ERROR: kubectl not found. Install: curl -LO https://dl.k8s.io/release/\$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl && chmod +x kubectl && sudo mv kubectl /usr/local/bin/kubectl"; exit 1; }

echo "Checking Podman rootful mode..."
ROOTFUL=$(podman machine inspect --format '{{.Rootful}}' 2>/dev/null || echo "unknown")
echo "Podman rootful: $ROOTFUL"
if [[ "$ROOTFUL" != "true" ]]; then
    echo "WARNING: Podman machine may not be rootful. kind requires rootful mode."
    echo "Fix: podman machine stop && podman machine set --rootful && podman machine start"
    echo "Continuing anyway — kind will fail if rootful is truly required..."
fi
echo ""

echo "=== Step 2: Create kind cluster ==="
export KIND_EXPERIMENTAL_PROVIDER=podman
kind create cluster --name "$CLUSTER_NAME" --wait 120s
echo "Cluster created successfully."
echo ""

echo "=== Step 3: Verify node is Ready ==="
kubectl wait --for=condition=Ready node/"$CLUSTER_NAME-control-plane" --timeout=120s
kubectl get nodes -o wide
echo ""

echo "=== Step 4: Check kube-system pods (CoreDNS health) ==="
kubectl get pods -n kube-system
echo ""

echo "=== Step 5: Deploy test pod ==="
kubectl run dns-test --image=busybox:1.36 --restart=Never -- sleep 300
echo "Waiting for pod to be Ready..."
kubectl wait --for=condition=Ready pod/dns-test --timeout=120s
echo "Pod is running."
echo ""

echo "=== Step 6: DNS validation ==="
echo "Resolving kubernetes.default..."
kubectl exec dns-test -- nslookup kubernetes.default
echo ""
echo "Resolving kubernetes.default.svc.cluster.local..."
kubectl exec dns-test -- nslookup kubernetes.default.svc.cluster.local
echo ""

echo "================================================================"
echo "  SPIKE PASSED"
echo ""
echo "  Results:"
echo "    - kind cluster: created via Podman ($ROOTFUL rootful)"
echo "    - Node status: Ready"
echo "    - Pod deployment: successful"
echo "    - DNS resolution: working"
echo ""
echo "  Phase 3 prerequisite: SATISFIED"
echo "================================================================"
```

Make executable: `chmod +x scripts/kind-smoke-test.sh`.

Then **run the script:** `bash scripts/kind-smoke-test.sh`.

The script will:
1. Verify kind, kubectl, and Podman rootful prerequisites
2. Create a single-node kind cluster named `bankforge-spike` using Podman
3. Wait for the node to reach Ready state
4. Check CoreDNS pods are healthy in kube-system
5. Deploy a BusyBox test pod
6. Run `nslookup kubernetes.default` and `nslookup kubernetes.default.svc.cluster.local` from inside the pod
7. Clean up the cluster (via EXIT trap, always runs)

**If the spike FAILS:**
- If kind cluster creation fails: Check `podman machine inspect` for rootful status. Try `podman machine stop && podman machine set --rootful && podman machine start` and retry.
- If DNS resolution fails: Check CoreDNS pod logs: `kubectl logs -n kube-system -l k8s-app=kube-dns`. If CoreDNS is in CrashLoopBackOff, this is the rootless Podman issue (Pitfall 6). Confirm rootful mode and retry.
- Document the failure mode and workaround in STATE.md.

**Acceptance criteria:**
- `scripts/kind-smoke-test.sh` exists and is executable
- Running the script outputs `SPIKE PASSED` (or failure is documented)
- No `bankforge-spike` kind cluster remains after the script completes (cleanup trap fired)
- `kind get clusters` does NOT list `bankforge-spike` after execution
