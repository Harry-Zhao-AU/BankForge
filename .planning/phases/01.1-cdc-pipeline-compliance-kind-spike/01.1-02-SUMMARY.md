# Phase 1.1 Plan 02: AUSTRAC Threshold Audit Logging Summary

**One-liner:** AUSTRAC-compliant threshold detection via dedicated Kafka consumer group with BigDecimal.compareTo() threshold check and named SLF4J logger (additivity=false) routing to separate log channel.

## What Was Built

Three artifacts implement AUSTRAC threshold reporting for the notification-service:

1. **`AustracAuditListener.java`** — Kafka consumer (`groupId=austrac-audit`) monitoring `banking.transfer.events`. For every transfer >= AUD $10,000 (checked via `compareTo()`, not `equals()`), logs a structured audit event to the `AUSTRAC_AUDIT` named logger with fields: `austrac_event`, `threshold_aud`, `transfer_id`, `from_account_id`, `to_account_id`, `amount`, `currency`, `event_timestamp`. Failed messages route to DLT via `@RetryableTopic`.

2. **`logback-spring.xml`** — Logback configuration adding an `AUSTRAC_FILE` rolling file appender (LogstashEncoder for structured JSON) and an `AUSTRAC_AUDIT` logger with `additivity="false"` so audit events never appear in operational console logs. The file path is configurable via `log.austrac.path` spring property.

3. **`AustracAuditListenerTest.java`** — Integration test with Testcontainers Kafka. Uses a logback `ListAppender` attached to the `AUSTRAC_AUDIT` logger at test setup to capture events in-memory. Test 1 verifies a $15,000 transfer produces an event with `austrac_event=true` and correct `transfer_id`. Test 2 verifies a $5,000 transfer produces no event (assertion holds for 5s). Both pass.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Jackson import: com.fasterxml -> tools.jackson**
- **Found during:** Task 02 (writing AustracAuditListener)
- **Issue:** Plan template used `com.fasterxml.jackson.databind.ObjectMapper/JsonNode` (Jackson 2.x) but project uses Jackson 3.x (`tools.jackson.databind`) as part of Spring Boot 4 / Spring Framework 7
- **Fix:** Used `tools.jackson.databind.JsonNode` and `tools.jackson.databind.ObjectMapper` in AustracAuditListener
- **Files modified:** `AustracAuditListener.java`
- **Commit:** 530986d

**2. [Rule 1 - Bug] @Backoff import removed in Spring Kafka 4.0**
- **Found during:** Task 02 compilation
- **Issue:** `org.springframework.retry.annotation.Backoff` does not exist — spring-retry is not a dependency of spring-kafka 4.0.4. Spring Kafka 4.0 introduced its own `org.springframework.kafka.annotation.BackOff` annotation with camelCase `backOff` attribute (not `backoff`)
- **Fix:** Changed import to `org.springframework.kafka.annotation.BackOff` and attribute name to `backOff` in both `AustracAuditListener.java` and `NotificationEventListener.java` (pre-existing bug)
- **Files modified:** `AustracAuditListener.java`, `NotificationEventListener.java`
- **Commit:** 530986d

**3. [Rule 1 - Bug] @SpyBean removed in Spring Boot 4**
- **Found during:** Task 03 test compilation
- **Issue:** `org.springframework.boot.test.mock.bean.SpyBean` does not exist in Spring Boot 4. Replaced with `org.springframework.test.context.bean.override.mockito.MockitoSpyBean`
- **Fix:** Updated import and annotation in `NotificationEventListenerIT.java`
- **Files modified:** `NotificationEventListenerIT.java`
- **Commit:** 7310e65

**4. [Rule 1 - Bug] KafkaContainer uses ZooKeeper wait strategy — incompatible with apache/kafka KRaft**
- **Found during:** Task 03 test execution (container launch failure)
- **Issue:** `org.testcontainers.containers.KafkaContainer` (deprecated) waits for `[KafkaServer id=\d+] started` which is a ZooKeeper-era log pattern. `apache/kafka:3.9.2` runs KRaft and never emits this message
- **Fix:** Switched to `org.testcontainers.kafka.KafkaContainer` (new, KRaft-aware) in both `AustracAuditListenerTest` and `NotificationEventListenerIT`
- **Files modified:** `AustracAuditListenerTest.java`, `NotificationEventListenerIT.java`
- **Commit:** 7310e65

**5. [Rule 2 - Missing] spring-boot-starter-kafka required for KafkaTemplate auto-config**
- **Found during:** Task 03 test execution (NoSuchBeanDefinitionException for KafkaTemplate)
- **Issue:** `spring-kafka` alone does not provide auto-configuration in Spring Boot 4. `KafkaTemplate` is only auto-configured when `spring-boot-starter-kafka` (which includes the auto-configure module) is declared
- **Fix:** Replaced `spring-kafka` with `spring-boot-starter-kafka` in notification-service pom.xml
- **Files modified:** `notification-service/pom.xml`
- **Commit:** 7310e65

**6. [Rule 3 - Blocking] Test design: logback springProperty cannot read @DynamicPropertySource**
- **Found during:** Task 03 test execution (audit log file not created in @TempDir)
- **Issue:** The plan specified file-based assertions using `@TempDir` + `@DynamicPropertySource` setting `log.austrac.path`. However, logback initializes before Spring dynamic properties are registered in `@SpringBootTest`, so `<springProperty>` reads the default path not the temp dir path
- **Fix:** Switched test assertions to use logback `ListAppender<ILoggingEvent>` attached to the `AUSTRAC_AUDIT` logger via `@BeforeEach`/`@AfterEach`. This captures in-memory events including `StructuredArguments.kv()` pairs. Behavioral verification (threshold detection) is fully preserved
- **Files modified:** `AustracAuditListenerTest.java`
- **Commit:** 7310e65
