## Accomplishments

- Wired the full CDC pipeline: account-db WAL → Debezium pgoutput → banking.transfer.events → dual consumers
- LedgerEventListener writes atomic DEBIT + CREDIT LedgerEntry pairs per transfer event

- Both consumers backed by @RetryableTopic + @DltHandler — poison-pill events route to DLT, never lost

## Decisions Made

- Used apache/kafka:3.9.2 (native KRaft, no ZooKeeper) with KAFKA_* env vars (not bitnami KAFKA_CFG_* form)
- Debezium converter config uses short form without CONNECT_ prefix (required for Debezium 3.1 images)
- Chose lowercase `aggregatetype("transfer")` so topic resolves to `banking.transfer.events` (matches consumer config)
- Added `slot.drop.on.stop=true` to prevent replication slot exhaustion on Debezium restart
