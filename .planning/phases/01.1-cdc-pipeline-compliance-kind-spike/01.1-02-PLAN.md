# Plan 01.1-02: AUSTRAC Threshold Audit Logging

## Objective

Implement a Kafka consumer in notification-service that monitors the `banking.transfer.events` topic for transfers >= AUD $10,000 and logs structured JSON audit events to a dedicated file. This satisfies Australian Transaction Reports and Analysis Centre (AUSTRAC) threshold reporting requirements.

Purpose: Financial compliance requires that all high-value transfers produce a queryable, structured audit trail separate from operational logs.

## Must-Haves (Goal-Backward Verification)

- [ ] `AustracAuditListener.java` exists with `@KafkaListener(topics = "banking.transfer.events", groupId = "austrac-audit")`
- [ ] `AustracAuditListener.java` compares transfer amount to `BigDecimal("10000.00")` threshold
- [ ] `AustracAuditListener.java` uses `LoggerFactory.getLogger("AUSTRAC_AUDIT")` (NOT the class-level @Slf4j logger)
- [ ] `logback-spring.xml` defines appender `AUSTRAC_FILE` routing to `logs/austrac-audit.log` with `LogstashEncoder`
- [ ] `logback-spring.xml` defines logger `AUSTRAC_AUDIT` with `additivity="false"` (no operational log mixing)
- [ ] `AustracAuditListenerTest.java` passes: $10,000 transfer produces audit entry; $9,999.99 transfer does not
- [ ] `mvn test -pl notification-service -Dtest=AustracAuditListenerTest -q` exits 0

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Log injection via payload content (Tampering) | MEDIUM | LogstashEncoder outputs structured JSON; values are JSON-escaped, not string-interpolated into log message templates |
| AUSTRAC audit events lost due to consumer failure (Denial of Service) | HIGH | `@RetryableTopic` with DLT routing; DLT topic `banking.transfer.events-dlt` preserves unprocessable events for manual review |
| Threshold bypass via BigDecimal comparison error (Information Disclosure) | MEDIUM | Use `compareTo()` (not `equals()`) for BigDecimal comparison; `10000.00` vs `10000.0000` must both trigger |
| Audit log capacity exhausted causes silent event loss (Repudiation) | MEDIUM | `maxHistory=365` with `totalSizeCap=5GB` in logback rolling policy prevents unbounded growth; add monitoring alert on `logs/austrac-audit.log` size in production |

### Task 01 — Create logback-spring.xml with AUSTRAC_AUDIT appender
**Wave:** 1

**Read first:**
- `notification-service/src/main/resources/application.yml` (current config)
- `notification-service/pom.xml` (confirm logstash-logback-encoder dependency from Plan 01)

**Action:**

Create `notification-service/src/main/resources/logback-spring.xml`. This file configures two appenders:
1. The default CONSOLE appender for operational logs (Spring Boot default pattern)
2. A dedicated `AUSTRAC_FILE` appender for compliance audit events

The audit log file path is configurable via property `log.austrac.path` (defaults to `logs/austrac-audit.log`). This allows integration tests to use `@DynamicPropertySource` to redirect audit output to a `@TempDir`, preventing cross-test state leakage.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <!-- Import Spring Boot default console appender -->
    <include resource="org/springframework/boot/logging/logback/defaults.xml"/>
    <include resource="org/springframework/boot/logging/logback/console-appender.xml"/>

    <!-- Configurable audit log path — overridable via Spring property log.austrac.path -->
    <springProperty scope="context" name="austracLogPath"
                    source="log.austrac.path"
                    defaultValue="logs/austrac-audit.log"/>

    <!-- AUSTRAC audit log — dedicated file, structured JSON, separate from operational logs -->
    <appender name="AUSTRAC_FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${austracLogPath}</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <fileNamePattern>${austracLogPath}.%d{yyyy-MM-dd}.%i</fileNamePattern>
            <maxFileSize>50MB</maxFileSize>
            <maxHistory>365</maxHistory>
            <totalSizeCap>5GB</totalSizeCap>
        </rollingPolicy>
        <encoder class="net.logstash.logback.encoder.LogstashEncoder"/>
    </appender>

    <!--
        AUSTRAC_AUDIT logger: additivity=false ensures audit events go ONLY to the
        dedicated file, not to the console or root logger. This is a compliance
        requirement — operational and audit logs must be separable.
    -->
    <logger name="AUSTRAC_AUDIT" level="INFO" additivity="false">
        <appender-ref ref="AUSTRAC_FILE"/>
    </logger>

    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
    </root>
</configuration>
```

Key design decisions:
- `additivity="false"` prevents AUSTRAC events from appearing in operational console logs (per D-08).
- `maxHistory=365` retains audit logs for 1 year (compliance best practice).
- `totalSizeCap=5GB` prevents unbounded disk growth (Threat Model: audit log capacity exhaustion).
- `LogstashEncoder` produces structured JSON with `@timestamp`, `logger_name`, `level`, and all key-value pairs from the log event.
- `springProperty` with `source="log.austrac.path"` allows tests to override the log path via `@DynamicPropertySource`.

**Acceptance criteria:**
- `notification-service/src/main/resources/logback-spring.xml` exists
- `grep "AUSTRAC_AUDIT" notification-service/src/main/resources/logback-spring.xml` returns matches
- `grep 'additivity="false"' notification-service/src/main/resources/logback-spring.xml` returns a match
- `grep "LogstashEncoder" notification-service/src/main/resources/logback-spring.xml` returns a match
- `grep "austracLogPath" notification-service/src/main/resources/logback-spring.xml` returns a match

```java
package au.com.bankforge.notification.kafka;

import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.kafka.annotation.DltHandler;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.annotation.RetryableTopic;
import org.springframework.kafka.support.KafkaHeaders;
import org.springframework.messaging.handler.annotation.Header;
import org.springframework.messaging.handler.annotation.Payload;
import org.springframework.retry.annotation.Backoff;
import org.springframework.stereotype.Component;

import java.math.BigDecimal;
import java.time.Instant;
import java.util.UUID;

import static net.logstash.logback.argument.StructuredArguments.kv;

/**
 * AUSTRAC threshold detection listener.
 *
 * Monitors all transfer events. When transfer amount >= AUD $10,000,
 * logs a structured JSON audit event to the AUSTRAC_AUDIT logger.
 *
 * Uses a SEPARATE consumer group ("austrac-audit") from the notification
 * consumer ("notification-service") so each receives every message independently.
 *
 * NOTE: The DLT topic "banking.transfer.events-dlt" is shared with
 * NotificationEventListener and LedgerEventListener. Each consumer group
 * produces its own DLT entries independently — a message failing in
 * austrac-audit does NOT affect the notification-service or ledger-service
 * consumer groups. Spring Kafka @RetryableTopic creates per-group retry
 * topics but shares the base DLT topic name by convention.
 */
@Component
@Slf4j
@RequiredArgsConstructor
public class AustracAuditListener {

    private static final BigDecimal THRESHOLD = new BigDecimal("10000.00");
    private static final Logger AUSTRAC_LOG = LoggerFactory.getLogger("AUSTRAC_AUDIT");

    private final ObjectMapper objectMapper;

    @RetryableTopic(
        attempts = "4",
        backoff = @Backoff(delay = 1000, multiplier = 2.0, maxDelay = 10000),
        dltTopicSuffix = "-dlt",
        autoCreateTopics = "true",
        numPartitions = "1",
        replicationFactor = "1"
    )
    @KafkaListener(
        topics = "banking.transfer.events",
        groupId = "austrac-audit"
    )
    public void onTransferEvent(
            @Payload String payload,
            @Header(KafkaHeaders.RECEIVED_TOPIC) String topic,
            @Header(KafkaHeaders.OFFSET) long offset) {

        JsonNode root = parsePayload(payload);

        BigDecimal amount = new BigDecimal(root.get("amount").asText());

        // Use compareTo, NOT equals — BigDecimal.equals considers scale:
        // new BigDecimal("10000.00").equals(new BigDecimal("10000.0000")) returns FALSE
        // new BigDecimal("10000.00").compareTo(new BigDecimal("10000.0000")) returns 0
        if (amount.compareTo(THRESHOLD) >= 0) {
            String transferId = root.get("transferId").asText();
            String fromAccountId = root.get("fromAccountId").asText();
            String toAccountId = root.get("toAccountId").asText();

            AUSTRAC_LOG.info("AUSTRAC threshold transfer detected",
                    kv("austrac_event", true),
                    kv("threshold_aud", THRESHOLD),
                    kv("transfer_id", transferId),
                    kv("from_account_id", fromAccountId),
                    kv("to_account_id", toAccountId),
                    kv("amount", amount.toPlainString()),
                    kv("currency", "AUD"),
                    kv("event_timestamp", Instant.now().toString()));

            log.info("AUSTRAC threshold triggered: transferId={} amount={}", transferId, amount);
        }
    }

    @DltHandler
    public void handleDlt(
            @Payload String payload,
            @Header(KafkaHeaders.RECEIVED_TOPIC) String topic) {
        log.error("AUSTRAC AUDIT DLT: topic={} payload={}", topic, payload);
    }

    private JsonNode parsePayload(String payload) {
        try {
            return objectMapper.readTree(payload);
        } catch (Exception e) {
            throw new RuntimeException("Failed to parse transfer event payload for AUSTRAC audit", e);
        }
    }
}
```

Key implementation details:
- Uses `LoggerFactory.getLogger("AUSTRAC_AUDIT")` — NOT the class-level `@Slf4j` log. The AUSTRAC_AUDIT logger routes to the dedicated file appender.
- Uses `net.logstash.logback.argument.StructuredArguments.kv()` to add structured fields to the JSON log entry. These appear as top-level JSON keys in the LogstashEncoder output.
- Uses `compareTo()` not `equals()` for BigDecimal threshold comparison (Threat Model row 3).
- Consumer group `austrac-audit` is separate from `notification-service` and `ledger-service`, so all three groups receive every message independently (TXNS-03 choreography pattern).
- The shared DLT topic `banking.transfer.events-dlt` is documented in the class Javadoc (see note above).

**Acceptance criteria:**
- `notification-service/src/main/java/au/com/bankforge/notification/kafka/AustracAuditListener.java` exists
- `grep "AUSTRAC_AUDIT" notification-service/src/main/java/au/com/bankforge/notification/kafka/AustracAuditListener.java` returns a match
- `grep "compareTo" notification-service/src/main/java/au/com/bankforge/notification/kafka/AustracAuditListener.java` returns a match
- `grep "austrac-audit" notification-service/src/main/java/au/com/bankforge/notification/kafka/AustracAuditListener.java` returns a match (groupId)

### Task 03 — Integration test for AUSTRAC threshold detection
**Wave:** 2 (depends on Task 01 logback + Task 02 listener)

**Read first:**
- `notification-service/src/test/java/au/com/bankforge/notification/kafka/NotificationEventListenerIT.java` (test pattern from Plan 01)
- `notification-service/src/main/java/au/com/bankforge/notification/kafka/AustracAuditListener.java` (implementation from Task 02)

**Action:**

Create `notification-service/src/test/java/au/com/bankforge/notification/kafka/AustracAuditListenerTest.java`:

This test verifies two behaviors:
1. A transfer of $15,000 (above threshold) produces an AUSTRAC audit log entry
2. A transfer of $5,000 (below threshold) does NOT produce an AUSTRAC audit log entry

The test uses Testcontainers Kafka, publishes events, waits for consumption with Awaitility, then reads the audit log file to verify structured JSON entries. Each test class gets an isolated audit log file via `@TempDir` and `@DynamicPropertySource` setting `log.austrac.path`, preventing cross-test state leakage. Tests are ordered to ensure above-threshold runs first, establishing the log file.

```java
package au.com.bankforge.notification.kafka;

import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;
import org.junit.jupiter.api.MethodOrderer;
import org.junit.jupiter.api.Order;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.TestMethodOrder;
import org.junit.jupiter.api.io.TempDir;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.test.context.DynamicPropertyRegistry;
import org.springframework.test.context.DynamicPropertySource;
import org.testcontainers.containers.KafkaContainer;
import org.testcontainers.junit.jupiter.Container;
import org.testcontainers.junit.jupiter.Testcontainers;
import org.testcontainers.utility.DockerImageName;

import java.nio.file.Files;
import java.nio.file.Path;
import java.util.List;
import java.util.UUID;
import java.util.concurrent.TimeUnit;

import static org.assertj.core.api.Assertions.assertThat;
import static org.awaitility.Awaitility.await;

@Testcontainers
@SpringBootTest
@TestMethodOrder(MethodOrderer.OrderAnnotation.class)
class AustracAuditListenerTest {

    @TempDir
    static Path tempDir;

    @Container
    static KafkaContainer kafka = new KafkaContainer(
            DockerImageName.parse("apache/kafka:3.9.2")
                .asCompatibleSubstituteFor("confluentinc/cp-kafka")
    );

    @DynamicPropertySource
    static void props(DynamicPropertyRegistry registry) {
        registry.add("spring.kafka.bootstrap-servers", kafka::getBootstrapServers);
        registry.add("spring.flyway.enabled", () -> "false");
        registry.add("log.austrac.path", () -> tempDir.resolve("austrac-audit.log").toString());
    }

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    private final ObjectMapper objectMapper = new ObjectMapper();

    @Test
    @Order(1)
    void shouldLogAustracEventForTransferAboveThreshold() {
        UUID transferId = UUID.randomUUID();
        String payload = """
            {
              "transferId": "%s",
              "fromAccountId": "%s",
              "toAccountId": "%s",
              "amount": "15000.0000",
              "currency": "AUD"
            }
            """.formatted(transferId, UUID.randomUUID(), UUID.randomUUID());

        kafkaTemplate.send("banking.transfer.events", transferId.toString(), payload);

        // Wait for the AUSTRAC listener to consume and log
        Path auditLog = tempDir.resolve("austrac-audit.log");
        await().atMost(30, TimeUnit.SECONDS).untilAsserted(() -> {
            assertThat(auditLog).exists();
            List<String> lines = Files.readAllLines(auditLog);
            boolean found = lines.stream().anyMatch(line -> {
                try {
                    JsonNode node = objectMapper.readTree(line);
                    return node.has("transfer_id")
                            && node.get("transfer_id").asText().equals(transferId.toString())
                            && node.has("austrac_event")
                            && node.get("austrac_event").asBoolean();
                } catch (Exception e) {
                    return false;
                }
            });
            assertThat(found)
                    .as("AUSTRAC audit log should contain entry for transfer %s", transferId)
                    .isTrue();
        });
    }

    @Test
    @Order(2)
    void shouldNotLogAustracEventForTransferBelowThreshold() {
        UUID transferId = UUID.randomUUID();
        String payload = """
            {
              "transferId": "%s",
              "fromAccountId": "%s",
              "toAccountId": "%s",
              "amount": "5000.0000",
              "currency": "AUD"
            }
            """.formatted(transferId, UUID.randomUUID(), UUID.randomUUID());

        kafkaTemplate.send("banking.transfer.events", transferId.toString(), payload);

        // Use Awaitility to wait long enough for consumer processing,
        // then assert no audit entry exists for this below-threshold transfer.
        // We wait a reasonable time to confirm absence, not just check immediately.
        await().atMost(30, TimeUnit.SECONDS).during(5, TimeUnit.SECONDS).untilAsserted(() -> {
            Path auditLog = tempDir.resolve("austrac-audit.log");
            if (auditLog.toFile().exists()) {
                List<String> lines = Files.readAllLines(auditLog);
                boolean found = lines.stream().anyMatch(line -> {
                    try {
                        JsonNode node = objectMapper.readTree(line);
                        return node.has("transfer_id")
                                && node.get("transfer_id").asText().equals(transferId.toString());
                    } catch (Exception e) {
                        return false;
                    }
                });
                assertThat(found)
                        .as("AUSTRAC audit log should NOT contain entry for sub-threshold transfer %s", transferId)
                        .isFalse();
            }
            // If file does not exist, that also means no audit event was logged — correct behavior
        });
    }
}
```

Ensure `awaitility` dependency is in notification-service `pom.xml` (added in Plan 01 Task 02):
```xml

            <groupId>org.awaitility</groupId>
            <artifactId>awaitility</artifactId>
            <scope>test</scope>
        </dependency>
```

**Key test design decisions:**
- `@TestMethodOrder(MethodOrderer.OrderAnnotation.class)` with `@Order(1)` and `@Order(2)` ensures above-threshold runs first, establishing the log file before the below-threshold test reads it.
- `@TempDir` with `@DynamicPropertySource` setting `log.austrac.path` gives each test class an isolated log file path, preventing cross-test state leakage from a shared `logs/austrac-audit.log`.
- `logback-spring.xml` reads the path from `${log.austrac.path}` via `<springProperty>` (see Task 01).
- `spring.flyway.enabled=false` as a defensive measure since notification-service may not have a database configured.
- `await().atMost(30, SECONDS).during(5, SECONDS)` for the below-threshold test ensures the assertion holds continuously for 5 seconds, giving the consumer time to process before concluding absence.

**Acceptance criteria:**
- `notification-service/src/test/java/au/com/bankforge/notification/kafka/AustracAuditListenerTest.java` exists
- `mvn test -pl notification-service -Dtest=AustracAuditListenerTest -q` exits 0
- After running the above-threshold test, the temp dir audit log contains a JSON line with `"austrac_event":true` and the correct transfer_id
- After running the below-threshold test, no new AUSTRAC entry for the below-threshold transfer ID appears in the log
