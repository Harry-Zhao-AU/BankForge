<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host network -> Compose network | Observability service ports exposed to localhost (4318, 16686, 9090, 3100, 3000) |
| Services -> OTel Collector | Banking services push telemetry to collector over internal Compose DNS |
| OTel Collector -> Backends | Collector fans out to Jaeger, Prometheus, Loki over internal Compose DNS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | OTel Collector port 4318 exposed to host | accept | Local dev only; no external access. Phase 3 will restrict to Kubernetes internal network. |
| T-02-02 | Elevation of Privilege | Grafana anonymous admin enabled (`GF_AUTH_ANONYMOUS_ORG_ROLE: Admin`) | accept | Local dev only; `GF_AUTH_ANONYMOUS_ENABLED` must be removed when deployed to any shared environment. Documented for Phase 3 removal. |
| T-02-03 | Information Disclosure | Trace attributes may contain account IDs, transfer amounts | accept | Local dev environment with test data only. Production would require OTel Collector attribute filtering processor. |
| T-02-04 | Tampering | No authentication on Prometheus, Loki, or Jaeger APIs | accept | All on localhost-only Compose network. Phase 3 Kubernetes deployment adds Istio mTLS for internal traffic. |
| T-02-05 | Denial of Service | Jaeger in-memory storage unbounded | mitigate | `max_traces: 100000` configured in jaeger-config.yaml limits memory consumption. |
</threat_model>
