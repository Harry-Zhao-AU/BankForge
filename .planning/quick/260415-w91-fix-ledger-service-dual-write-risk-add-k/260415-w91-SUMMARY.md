# Quick Task 260415-w91: Fix ledger-service dual-write risk

**Completed:** 2026-04-15
**Commits:** 3bfe870, 24c9622

## What was done

Fixed the dual-write risk in `ledger-service` where a JVM crash between DB commit and `kafkaTemplate.send()` would leave the consumer offset unadvanced — causing ledger entries to be re-written on retry without any idempotency protection.

### Task 1 — DB idempotency guard (3bfe870)
- `V2__add_ledger_idempotency.sql`: added `UNIQUE (transfer_id, entry_type)` constraint to `ledger_entries`
- `LedgerEntryRepository.java`: added `existsByTransferId(UUID)` method

### Task 2 — Kafka EOS + idempotent listener (24c9622)
- `application.yml`: added `spring.kafka.producer.transaction-id-prefix: ledger-tx-` and `consumer.isolation-level: read_committed`
- `KafkaConfig.java` (new): `KafkaTransactionManager` with `EOSMode.V2`, `ConcurrentKafkaListenerContainerFactory` with `DefaultErrorHandler` (exponential backoff 1s→10s, max 3 retries); explicit `@Primary JpaTransactionManager` bean to resolve Spring Data qualifier conflict
- `LedgerEventListener.java`: removed `@RetryableTopic`/`@DltHandler` (conflicts with EOS producers); added `existsByTransferId()` guard — on replay, skips DB inserts but still publishes `banking.transfer.confirmed` within the Kafka TX
- `LedgerEventListenerIT.java`: added EOS dynamic props, `executeInTransaction` for sends, `isolation.level=read_committed` on verifier consumer, new `shouldBeIdempotentOnDuplicateDelivery` test

## Key design decisions

- `@RetryableTopic` removed: it creates non-transactional internal producers that conflict with Kafka EOS mode. `DefaultErrorHandler` on the container factory replaces it.
- Idempotency guard re-publishes confirmation on replay: downstream `TransferConfirmationListener` may not have received it if the original crash was after DB commit but before Kafka TX commit.
- `@Primary JpaTransactionManager` required: Spring Boot stops auto-registering the default JPA TM when a second `PlatformTransactionManager` bean exists — explicit declaration resolves qualifier ambiguity.
